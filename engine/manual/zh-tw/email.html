讓妳的電子郵件程式跟 POPFile 交談

POPFile 以代理伺服器的型態運作. 妳的電子郵件程式會跟 POPFile 交談, 而她自己會去跟妳的電子郵件伺服器交談. 於是妳的電子郵件程式將不再直接取得郵件訊, 而是讓 POPFile 先取得她們, 並且加以掃瞄來決定她們該被裝進那個郵筒裏.

一旦決定了之後 - 這祇會花零點幾秒而已 - POPFile 就可以再做下列兩件事: 他可以在主旨列最前面加上像 [這樣] 的標籤; 這表示 Microsoft Outlook Express 這類簡單的過濾程式也能把郵件轉發到她們該去的地方 - 或者, 對於像 Eudora 這類更進階的電子郵件程式來說, 她也能加入一列像這樣的新標頭:

X-Text-Classification: personal

- 而妳的電子郵件程式就可以依此來加以過濾.

然而, 妳得做兩件事纔能讓電子郵件程式去跟 POPFile 對話:

  1. 調整妳的電子郵件程式的設定好讓她去跟 POPFile 對話, 以及
  2. 設立妳的電子郵件過濾器來妥善處理這件事.

讓咱們先調整郵件設定. 稍後我們馬上就會設立過濾器了.

底下是給一些較常見電子郵件程式的指引. 對於其他的電子郵件軟體 - 或對進階使用者 - 來說, 以下為一般性的指引:

  • 把電子郵件用戶端的 POP3 伺服器名稱改為 127.0.0.1, 把原始的地址記下來.
  • 把 POP3 的使用者名稱改為 原來的-POP3-伺服器地址:使用者名稱
  • 不要去動密碼的部分

對於使用保全密碼認證 (SPA) 的帳號, 請參見搭配 POPFile 一併使用: 保全密碼認證 (SPA).

從底下選出屬於妳的電子郵件程式那一欄然後讀下去. 雖然這些指引看來複雜, 但其實兩分鐘內就可以做完了 - 而且還很難弄錯. (不過為了以防萬一, 妳最好還是把原有的更定值給記下來, 以備將來有需要把他們給設回去.)

Outlook

  • 在 Outlook 裏選擇 工具->郵件帳號... 選單選項. 接下來就會出現電子郵件帳號對話窗. [螢幕快照]
  • 按下 檢視或變更既有的電子郵件帳號 然後按 下一步. 選擇妳想使用 POPFile 的帳號然後按下 變更... 把內收郵件伺服器 (POP3) 的伺服器名稱跟使用者名稱記下來. [螢幕快照]
  • 把內收郵件伺服器 (POP3) 的伺服器名稱改成 127.0.0.1 然後, 把使用者名稱改成原來的內收郵件伺服器 (POP3) 的伺服器名稱連著原來的使用者名稱, 並以冒號隔開. [螢幕快照]
  • 所以, 如果原來的內收郵件伺服器 (POP3) 名稱為 my.mail.com 而使用者名稱為 jdoe 的話, 新的使用者名稱就會是 my.mail.com:jdoe
  • 妳並不需要變更密碼.
  • 按 下一步 然後按 完成.
  • 確認 POPFile 正在執行, 而所有的電子郵件都會透過 POPFile 寄達!
  • 妳得確定任何 Widows 啟動的時候 POPFile 也都一併啟動了. 如果妳正在使用 Windows 安裝版的話, 可以把 'Start POPFile in Background' 鏈結從 POPFile 群組裏拖曳到開始選單的啟動群組裏. 那麼, 每次 Windows 都將會載入 POPFile 了.
  • 重要! 如果 POPFile 沒有被載入的話, 郵件就不能通過了 - 而妳的電子郵件程式將無法連線到郵件伺服器!

Outlook Express

  • 在 Outlook Express 裏選擇 工具->帳號... 選單選項. 接下來就會出現網際網路帳號對話窗. [螢幕快照]
  • 選擇妳想要修改來使用 POPFile 的帳號然後在 內容 上按一下. [螢幕快照]
  • 選擇 伺服器 分頁. 把內收郵件 (POP3) 的伺服器名稱跟內收郵件伺服器帳號名稱記下來. [螢幕快照]
  • 把內收郵件 (POP3) 的伺服器名稱改成 127.0.0.1 然後, 把內收郵件伺服器帳號名稱 改成原來的內收郵件 (POP3) 伺服器名稱後面接著原來的內收郵件伺服器帳號名稱, 並用冒號隔開. [螢幕快照]
  • 所以, 如果原來的內收郵件伺服器名稱是 my.mail.com 而妳的使用者名稱為 jdoe 的話, 新的使用者名稱就會是 my.mail.com:jdoe
  • 妳並不需要變更密碼.
  • 按一下 OK 然後按 關閉.
  • 確認 POPFile 正在執行, 而所有的電子郵件都會透過 POPFile 寄達!
  • 妳得確定任何 Widows 啟動的時候 POPFile 也都一併啟動了. 如果妳正在使用 Windows 安裝版的話, 可以把 'Start POPFile in Background' 鏈結從 POPFile 群組裏拖曳到開始選單的啟動群組裏. 那麼, 每次 Windows 都將會載入 POPFile 了.
  • 重要! 如果 POPFile 沒有被載入的話, 郵件就不能通過了 - 而妳的電子郵件程式將無法連線到郵件伺服器!

Eudora

  • 在 Eudora 裏選擇 工具->選項... 選單選項. 接下來會出現選項對話窗. [螢幕快照]
  • 把郵件伺服器 (內送) 的伺服器名稱跟登入名稱記下來. [螢幕快照]
  • 把郵件伺服器 (內送) 的伺服器名稱改為 127.0.0.1 然後, 把登入名稱改成原來的郵件伺服器 (內送) 伺服器名稱後面接著原來的登入名稱, 並以冒號分隔. [螢幕快照]
  • 所以, 如果原來的郵件伺服器 (內送) 伺服器名稱是 my.mail.com 而使用者名稱為 jdoe 的話, 新的使用者名稱就會是 my.mail.com:jdoe
  • 妳並不需要變更密碼.
  • 按一下 OK.
  • 確認 POPFile 正在執行, 而所有的電子郵件都會透過 POPFile 寄達!
  • 妳得確定任何 Widows 啟動的時候 POPFile 也都一併啟動了. 如果妳正在使用 Windows 安裝版的話, 可以把 'Start POPFile in Background' 鏈結從 POPFile 群組裏拖曳到開始選單的啟動群組裏. 那麼, 每次 Windows 都將會載入 POPFile 了.
  • 重要! 如果 POPFile 沒有被載入的話, 郵件就不能通過了 - 而妳的電子郵件程式將無法連線到郵件伺服器!

Pegasus

  • 在 Pegasus 裏選擇 工具->網際網路選項... 接收 (POP3) 分頁. 接下來會出現 POP3 帳號設定頁面.
  • 在 使用 POP3 接收信件的一邊性設定 裏, 選擇 POP3 主機 對話盒. 把目前的 POP3 主機記下來.
  • 把 POP3 主機改為 127.0.0.1
  • 在 使用 POP3 接收信件的一邊性設定 裏, 選擇 使用者名稱 對話盒. 把目前的使用者名稱記下來.
  • 把使用者名稱改成原來的 POP3 主機名稱後面接著原來的使用者名稱, 並以冒號分隔.
  • 所以, 如果原來的 POP3 主機是 my.mail.com 而使用者名稱是 jdoe 的話, 新的使用者名稱就會是 my.mail.com:jdoe
  • 妳並不需要變更密碼.
  • 按一下 OK 按鈕來儲存變更.
  • 確認 POPFile 正在執行, 而所有的電子郵件都會透過 POPFile 寄達!
  • 妳得確定任何 Widows 啟動的時候 POPFile 也都一併啟動了. 如果妳正在使用 Windows 安裝版的話, 可以把 'Start POPFile in Background' 鏈結從 POPFile 群組裏拖曳到 開始 -> 程式 -> 啟動 群組裏. 那麼, 每次 Windows 都將會載入 POPFile 了.

好. 現在我們完成了上述的步驟, 該是要來設立郵件過濾器的時候了, 這麼一來當 POPFile 完成工作後, 妳的電子郵件程式纔能把每一封信歸到正確的信件匣裏.

對於進階的使用者來說, 妳可以決定是否要變更主旨列來加入 [分類] 標籤. 這個選項在網頁介面中組態分頁接近最底下的地方. 不論妳是否選擇了這個選項, 都會加入 X-Text-Classification 標頭.

Outlook

  • 在 Outlook 裏選擇 工具->規則精靈... [螢幕快照]
  • 接下來會出現規則精靈. 按一下 新增... 然後選擇 從空白規則開始. [螢幕快照]
  • Outlook 可以使用 X-Text-Classification 標頭, 這麼一來郵件過濾就會完全在幕後完成. 按一下 下一步 然後選擇 用訊息標頭裏的特定文字.
  • 現在按一下灰色高亮度的特定文字鏈結, 然後在出現的對話盒裏輸入 "X-Text-Classification: " 後面接著妳的郵筒名稱, 然後按下 加入, 再按 OK.
  • 按一下 下一步 然後選擇移動到指定的信件匣.
  • 按一下灰色高亮度的指定的文字, 然後會出現一份信件匣清單. 選擇任何妳想要移動該郵筒內的信件的目標信件匣, 然後按 OK.
  • 按一下 OK, 然後按 完成.
  • 重複上述的步驟, 直到每一個郵筒都設定完畢, 然後按 OK 離開.
  • 大功告成!
  • 提醒妳: 在 POPFile 網頁介面中, 在組態分頁底下, 檢查一下要把修改主旨列選項關閉.

Outlook Express

  • 在 Outlook Express 裏選擇 工具->訊息規則->郵件... [螢幕快照]
  • 接下來會出現新增郵件規則視窗. [螢幕快照]
  • Outlook Express 並不能使用 X-Text-Classification 標頭, 所以就不能完全在幕後過濾郵件了. 於是, 請改按主旨含有特定文字然後按下移動到特定的信件匣 [螢幕快照]
  • 現在按一下藍色高亮度的含有特定文字鏈結, 然後會出現鍵入特定文字的對話窗 [螢幕快照]
  • 輸入一個郵筒名稱, 並且以 [方括號] 包住, 然後按 OK. 現在按一下藍色高亮度的特定, 接下來會出現移動對話窗. 按一下妳想要從郵筒移動訊息到的目標信件匣, 然後按 OK. [螢幕快照]
  • 按一下 OK.
  • 重複上述的步驟, 直到每一個郵筒都設定完畢, 然後按 OK 離開.
  • 大功告成!
  • 提醒妳: 在 POPFile 網頁介面中, 在組態分頁底下, 檢查一下要把修改主旨列選項打開.

Eudora

Eudora 可以用任何郵件標頭來過濾郵件, 這實在太棒了 - 所有的工作都可以完全在背景完成, 而不會變更主旨列!

  • 選擇 工具->過濾器 選單. [螢幕快照]
  • 接下來會出現過濾器視窗. 按一下 新增 來建立新的過濾器. [螢幕快照]
  • 在標頭對話盒裏鍵入 X-Text-Classification 然後再下一個對話盒裏鍵入某個郵筒的名稱. 不要在周圍加上方括號. [螢幕快照]
  • 然後在第一個動作對話盒裏選擇傳送到, 然後選擇妳希望該郵筒內的信件所要遞送到的信箱. [螢幕快照]
  • 重複上述的步驟, 直到每一個郵筒都設定完畢, 然後按 OK 離開.
  • 關閉過濾器視窗, 當詢問是否要儲存變更時回答 是.
  • 大功告成!
  • 提醒妳: 在 POPFile 網頁介面中, 在組態分頁底下, 檢查一下要把修改主旨列選項關閉.

Pegasus

  • 在 Pegasus 裏選擇 工具->郵件過濾規則 ->編輯新郵件過濾規則 ->信件匣開啟時套用規則.
  • 接下來會出現新郵件過濾規則視窗.
  • 按一下 新規則 按鈕. 接下來會出現建立一個新的過濾規則視窗.
  • 在 表示式 按鈕上按一下. 請注意用表示式過濾器的話, 就算 POPFile 的修改主旨列選項被關閉了也能動. 如果妳要用標頭過濾器的話, 就一定要開啟 POPFile 的修改主旨列選項纔行.
  • 選擇 如果吻合此正規表示式 對話盒
  • 輸入某個郵筒的名稱, 前面加上 X-Text-Classification:, 像是: X-Text-Classification: spam
  • 選擇 標頭或本體 圓鈕
  • 從 動作 下拉式選單裏選擇想要做的事. 像是 移動
  • 如果被選擇了像是 移動 或 轉寄 的動作, 按下 設定 按鈕.
  • 選擇目標信件匣或者輸入要轉寄的地址以搭配所選的動作.
  • 按下 OK 按鈕來將規則加入到規則清單裏.
  • 重複上述的程序來建立所有妳所需的規則.
  • 過濾規則被套用的次序能夠用新郵件過濾規則視窗裏的紅/藍 (上/下) 箭頭加以調整.
  • 選擇 儲存 按鈕來儲存規則設定的變更.
  • 選擇 OK 按鈕.
  • 大功告成!

這就是整件事了!... 但是...

除此之外, POPFile 現在仍然不會過濾任何郵件. 為什麼不會呢? 因為她根本不知道妳所輸入的郵筒到底要做甚麼用. 她不曉得 '垃圾 (spam)' 和 '名符其實 (genuine)' 代表著甚麼, 也不知道 '工作 (work)' 跟 '私人 (personal)' 又是甚麼. 她甚至不曉得妳的電子郵件是以何種語言撰寫而成 - 事實上她根本就不在乎這件事. 妳得調教她把垃圾信從一般的電子郵件裏分出來, 或者把某個專案跟其他專案加以區隔. 妳可以調教她來做這件事 - 而我們即將開始這個工作.
