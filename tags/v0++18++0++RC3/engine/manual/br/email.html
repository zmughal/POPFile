Fazendo Seu Programa De Email Conversar Com O POPFile

O POPFile funciona como um servidor proxy. Seu programa de email conversa com o POPFile, que conversa com seu servidor de email. Ao invés de seu programa de email pegar as mensagens diretamente, o POPFile as pega primeiro, e as analisa para decidir em qual bucket elas devem ser colocadas.

Uma vez que isto for decidido - o que leva apenas uma fração de segundo - o POPFile faz uma das duas coisas: ele pode adicionar uma marcação como [esta] ao início da linha de assunto; o que significa que programas com filtragem simples como o Microsoft Outlook Express podem colocar seu email onde seria esperado - ou, para programas de email mais avançados como o Eudora, ele pode adicionar um novo cabeçalho ao email como este:

X-Text-Classification: pessoal

- o qual seu programa de email pode usar como um filtro.

De qualquer forma, você precisa fazer duas coisas para seu programa de email conversar com o POPFile:

  1. Ajustar as configurações do seu programa de email para fazê-lo conversar com o POPFile, e
  2. configurar seus filtros de email de maneira que os emails sejam filtrados corretamente.

Vamos ajustar as configurações de email primeiro. Vamos configurar os filtros em seguida.

Abaixo estão as instruções para alguns dos mais populares programas de email. Para outros - ou para usuários avançados - aqui estão as instruções gerais:

  • Troque o nome do servidor POP3 em seu cliente de email para 127.0.0.1, anotando o endereço original.
  • Troque o nome do usuário POP3 para [endereço-original-do-servidor-POP3]:[nome-do-usuário]
  • Deixe a senha como está

Para contas usando Autenticação Segura de Senha, veja Usando o POPFile com: Autenticação Segura de Senha.

Apenas escolha seu programa de email e siga a coluna abaixo. Apesar de estas instruções parecerem complicadas, na verdade elas levam apenas em torno de dois minutos - e é fácil acertar. (Só para garantir, faça uma anotação de suas configurações originais caso você precise restaurar algo.)

Outlook

  • No Outlook selecione o menu Ferramentas->Contas de Email... A janela Contas de Email vai aparecer. [screenshot]
  • Clique em Visualizar ou alterar uma conta de email existente e clique em Próximo. Escolha a conta que você quer usar com o POPFile e clique em Alterar... Anote o nome do servidor de entrada de email (POP3) e o nome do usuário. [screenshot]
  • Troque o nome do servidor de entrada de email (POP3) para 127.0.0.1 e troque o nome do usuário para uma combinação do nome do servidor de entrada original (POP3) e do nome do usuário original separados por dois pontos. [screenshot]
  • Assim, se o nome do servidor de entrada de email (POP3) original for my.mail.com e seu nome de usuário for jdoe, seu novo nome de usuário deverá ser my.mail.com:jdoe
  • Você não precisa mudar a senha.
  • Clique em Próximo e então em Finalizar.
  • Assegure-se de que o POPFile está rodando e todos os emails serão recebidos através do POPFile!
  • Você deverá se assegurar de que o POPFile inicie sempre ao iniciar o Windows. Se você usou a versão com instalador para Windows, você pode arrastar o atalho 'Start POPFile in Background' do grupo POPFile no menu Iniciar para o grupo Iniciar. Agora, o Windows irá carregar o POPFile cada vez que iniciar.
  • Importante! Se o POPFile não for carregado, os emails não serão recebidos - e o seu programa de email não vai conectar!

Pegasus

  • No Pegasus selecione o menu Tools->Internet Options... Receiving (POP3). A página de configuração de contas POP3 vai aparecer.
  • Em Configurações gerais para recebimento de email usando POP3, selecione POP3 Host. Anote host POP3 atual.
  • Troque o POP3 Host para 127.0.0.1
  • Em Configurações gerais para recebimento de email usando POP3, selecione User name. Anote o User name atual.
  • Troque o User name para uma combinação do POP3 Host original e o User name original separados por dois pontos.
  • Assim, se o POP3 host original for my.mail.com e seu User name for jdoe, seu novo User name será my.mail.com:jdoe
  • Você não precisa trocar a senha.
  • Clique no botão OK e salve as alterações.

Ok. Agora que fizemos isto, é hora de configurar seus filtros de email, assim uma vez que o POPFile tenha feito seu serviço, seu programa de email poderá colocar tudo na pasta correta.

Para usuários avançados, você tem a opção de modificar a linha de assunto para adicionar uma marca de [classificação] ou não. Isto está localizado pelo final da seção Configuração da interface web. Não importa qual opção você escolher, o cabeçalho X-Text-Classification sempre será adicionado.

Apenas escolha seu programa de email e siga a coluna abaixo. Apesar de estas instruções parecerem complicadas, na verdade elas levam apenas em torno de dois minutos - e é fácil acertar. (Só para garantir, faça uma anotação de suas configurações originais caso você precise restaurar algo.)

Outlook

  • No Outlook selecione Ferramentas->Assistente de Regras... [screenshot]
  • O Assistente de Regras aparece. Clique Nova... e selecione Iniciar com uma regra em branco. [screenshot]
  • O Outlook pode usar o cabeçalho X-Text-Classification, o que permite que a filtragem de email ocorra de forma completamente transparente. Clique em Próximo e selecione com palavras específicas no cabeçalho da mensagem.
  • Agora clique no link palavras específicas e digite "X-Text-Classification: " seguido pelo nome de um dos buckets e clique em Adicionar, e então clique em OK.
  • Clique em Próximo e selecione mover para a pasta especificada.
  • Clique na palavra especificada e uma lista de pastas aparece. Selecione qualquer pasta para a qual você quiser mover as mensagens daquele bucket e clique em OK.
  • Clique em OK, e então em Finalizar.
  • Repita o processo para cada um de seus buckets, e então clique em OK.
  • Feito!
  • Apenas uma pequena nota: na interface web do POPFile, na seção Configuração, verifique se a opção Modificação da Linha de Assunto está desligada.

Outlook Express

  • No Outlook Express selecione Ferramentas->Regras para Mensagens->Email... [screenshot]
  • A janela Regras para mensagens aparece. [screenshot]
  • O Outlook Express não pode usar o cabeçalho X-Text-Classification, o qual permite que a filtragem de email ocorra de forma completamente transparente. Assim ao invés disso, clique em Quando a linha 'Assunto' contiver palavras específicas e tambem clique em Movê-la para a pasta especificada [screenshot]
  • Agora clique no link contiver palavras específicas e a janela Digitar palavras específicas aparece [screenshot]
  • Digite o nome de um de seus buckets, entre [colchetes], e então clique em OK. Agora clique no link especificada e a janela Mover aparece. Clique na pasta para a qual você quer mover suas mensagens deste bucket e clique em OK. [screenshot]
  • Clique em OK.
  • Repita o processo para cada um de seus buckets, e então clique em OK.
  • Feito!
  • Apenas uma pequena nota: na interface web do POPFile, na seção Configuração, verifique se a opção Modificação da Linha de Assunto está ligada.

Eudora

O Eudora permite a filtragem baseada em qualquer cabeçalho de email, o que é ótimo - porque tudo pode ser feito de uma forma completamente transparente, sem alterar a linha de assunto!

  • Selecione o menu Tools->Filters. [screenshot]
  • A janela Filters aparece. Clique em New para criar um novo filtro. [screenshot]
  • Em Header digite X-Text-Classification e em contains digite o nome de um de seus buckets. Não adicione colchetes ao redor. [screenshot]
  • Selecine Transfer To para a primeira Action e selecione a caixa de email para a qual você quer que sejam transferidas as mensagens deste bucket. [screenshot]
  • Repita esta operação para cada um de seus buckets.
  • Feche a janela Filters e confirme com Yes quando perguntado sobre salvar as alterações.
  • Feito!
  • Apenas uma pequena nota: na interface web do POPFile, na seção Configuração, verifique se a opção Modificação da Linha de Assunto está desligada.

Pegasus

  • No Pegasus selecione Tools->Mail Filtering Rules ->Edit New Mail Filtering Rules ->Rules Applied When Folder is Opened.
  • A janela New Mail Filtering Rules aparece.
  • Clique no botão New Rule. A janela Create a New Filtering Rule aparece.
  • Clique no botão Expression. Note que usar o filtro Expression vai funcionar mesmo se a opção Modificação da Linha de Assunto do POPFile estiver desligada. Se você usar o filtro Header, a opção Modificação da Linha de Assunto do POPFile deve estar ligada.
  • Selecione If this regular expression
  • Digite o nome de um de seus buckets, precedido de X-Text-Classification:, por exemplo: X-Text-Classification: spam
  • Selecione Headers or body
  • Selecione a ação desejada na lista Action. Por exemplo, Move
  • Se uma ação como Move ou Forward foi escolhida, clique no botão Set.
  • Selecione a pasta de destino ou digite o endereço para encaminhar como for apropriado para a ação Action selecionada.
  • Clique no botão OK para adicionar a regra na lista de regras.
  • Repita o processo para cada regra que você precisar.
  • A ordem em que cada regra de filtro é aplicada pode ser ajustada usando as setas vermelha / azul (para cima / para baixo) da janela New Mail Filter Rules.
  • Selecione o botão Save para salvar o conjunto de alterações nas regras.
  • Selecione o botão OK.
  • Feito!

Está tudo certo!... mas...

Apesar de tudo isto, o POPFile ainda não filtra email. Porque não? Ele não sabe o que cada um dos buckets que você criou significam. Ele não tem idéia do que significa 'spam' ou 'confiável', ou o que é 'trabalho' ou 'pessoal'. Ele nem mesmo sabe em que língua estão escritos seus emails - e isso não importa. Você precisa treiná-lo para distinguir entre spam e email normal, ou entre um de seus projetos. Você faz isto através de treinamento - e nós vamos tratar disto a seguir.
