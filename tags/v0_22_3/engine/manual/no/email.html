Få ditt e-postprogram til å snakke med POPFile

POPFile virker som en proxy server. Ditt e-postprogram snakker med POPFile, som igjen snakker med e-postserveren på vegne av e-postprogrammet. Istedenfor at e-postprogrammet henter beskjedene direkte, henter POPFile dem først, og søker gjennom dem for så å avgjøre hvilken bøtte de skal i.

Så snart det er bestemt - noe som tar en brøkdel av et sekund - gjør POPFile en av to ting: et merke som [dette] kan legges til starten på emne-linjen; som betyr at  e-postprogrammer som er i stand til enkel e-postfiltrering (som Microsoft Outlook Express), kan levere e-posten i mapper etter ditt valg - eller, for mer avanserte e-postprogrammer som Eudora, kan POPFile legge til et nytt e-post hode (e-mail header) som for eksempel denne:

X-Text-Classification: personlig

- som ditt e-postprogram kan bruke som et filter.

Likevel, du må gjøre to ting for å få e-postprogrammet ditt til å snakke med POPFile:

  1. Justere e-postprogrammet ditt sine instillinger slik at det snakker med POPFile, og
  2. sette opp e-postfiltrene i e-postprogrammet slik at beskjeden blir sendt dit de skal.

La oss justere e-postinnstillingene først. Vi setter opp filtrene om et øyeblikk.

Nedenfor finner du instruksjonene for de mest populære programmene. For andre e-postprogrammer - eller for avanserte brukere - her følger de generelle instruksjonene:

  • Forandre navnet på POP3-serveren i din e-postklient til127.0.0.1. Legg merke til den opprinnelige adressen.
  • Forandre POP3 brukernavnet til [opprinnelig-POP3-server-adresse]:[brukernavn]
  • La passordet være i fred.

For kontoer som benytter sikker godkjenning av passord (Secure Password Authentication, SPA), se Bruk av POPFile med: Sikker Godkjenning av Passord (SPA).

Velg ditt e-postprogram og følg kolonnen helt ned. Selv om disse instruksjonene kan synes kompliserte, tar de faktisk bare et par minutter - og det er vanskelig å gjøre det feil. (Bare i tilfelle: Skriv ned de opprinnelige innstillingene etter hvert som du gjør endringer, slik at du lett kan gjøre det om igjen dersom noe skulle gå galt).

Outlook

  • I Outlook, klikk menyvalget Verktøy -> E-postkontoer...
    E-postkontoer-dialogen vil dukke opp [bilde]
  • Klikk 'Vise eller endre eksisterende e-postkontoer' og klikk 'Neste'.
  • Velg din konto, klikk så 'Endre...' Legg merke til (og noter)  Inkommende e-post servernavnet (POP3) og Brukernavnet. [bilde]
  • Forandre servernavnet for Innkommende e-post (POP3) til 127.0.0.1 og forandre Brukernavnet til en kombinasjon av det opprinnelige servernavnet for Inkommende e-post (POP3) og det opprinnelige Brukernavnet, atskilt med et kolon [bilde]
  • Så, hvis det opprinnelige servernavnet for Inkommende e-post (POP3) var min.epost.no og brukernavnet ditt var onordman, vil ditt nye brukernavn bli min.epost.no:onordman
  • Du trenger ikke forandre passord.
  • Klikk 'Neste' og så 'Fullfør'.
  • Forsikre deg om at POPFile kjører, så vil all e-post bli levert gjennom POPFile!
  • Du må sikre deg at POPFile starter hver gang du starter Windows.Hvis du installerte Windows-versjonen kan du dra 'Start POPFile in Background' linken fra POPFile gruppen i Startmenyen til Oppstart-gruppen, så vil POPFile starte hver gang Windows starter.
  • Viktig! Hvis POPFile ikke kjører vil ikke e-postprogrammet kunne koble til og du får ikke hentet e-post!

Outlook Express

  • Gå til menyen
    Verktøy -> Kontoer...  [bilde]
  • Velg kontoen din og klikk 'Egenskaper'. [bilde]
  • Velg arkfanen 'Servere' Legg merke til (og noter)  Inkommende e-post servernavnet (POP3) og Kontonavnet for innkommende epostserver. [bilde]
  • Forandre servernavnet for Innkommende e-post (POP3) til 127.0.0.1 og forandre Kontonavnet for innkommende epostserver til en kombinasjon av det opprinnelige servernavnet for Inkommende e-post (POP3) og det opprinnelige Kontonavnet for innkommende epostserver, atskilt med et kolon [bilde]
  • Så, hvis det opprinnelige servernavnet for Inkommende e-post (POP3) var min.epost.no og det opprinnelige kontonavnet onordman, vil ditt nye kontonavn bli min.epost.no:onordman
  • Du trenger ikke forandre passord.
  • Klikk 'OK' og så 'Lukk'.
  • Forsikre deg om at POPFile kjører, så vil all e-post bli levert gjennom POPFile! 
  • Du må sikre deg at POPFile starter hver gang du starter Windows.Hvis du installerte Windows-versjonen kan du dra 'Start POPFile in Background' linken fra POPFile gruppen i Startmenyen til Oppstart-gruppen, så vil POPFile starte hver gang Windows starter.
  • Viktig! Hvis POPFile ikke kjører vil ikke e-postprogrammet kunne koble til og du får ikke hentet e-post!

Eudora

(Merknad: Ettersom oversetter av manualen ikke benytter Eudora er disse instruksjonene fortsatt på engelsk)

  • In Eudora select the Tools->Options... menu option. The Options dialog will appear. [screenshot]
  • Make a note of the Mail Server (incoming) server name and the Login Name. [screenshot]
  • Change the Mail Server (incoming) server name to 127.0.0.1 and change the Login Name to the combination of the original Mail Server (incoming) server name and the original Login Name separated by a colon. [screenshot]
  • So, if the original Mail Server (incoming) server name was my.mail.com and your username was jdoe, your new username would be my.mail.com:jdoe
  • You don't need to change the password.
  • Hit OK.
  • Ensure that POPFile is running and all mail will be delivered through POPFile.
  • You'll need to make sure POPFile starts whenever Windows does. If you're using the Windows installer version, you can drag the 'Start POPFile in Background' link from the POPFile group in the Start menu to the Startup group. Now, Windows will load POPFile each time.
  • Important! If POPFile isn't loaded, the mail will not get through - and your email program won't connect!

Pegasus

(Merknad: Ettersom oversetter av manualen ikke benytter Pegasus er disse instruksjonene fortsatt på engelsk)

  • In Pegasus select the Tools->Internet Options... Receiving (POP3) tab. The POP3 account setup page will appear.
  • Under General settings for receiving mail using POP3, select the POP3 Host dialog box. Make a note of the current POP3 host.
  • Change the POP3 Host to 127.0.0.1
  • Under General settings for receiving mail using POP3, select the User name dialog box. Make a note of the current User name.
  • Change the User name to the combination of the original POP3 Host name and the original User name separated by a colon.
  • So, if the original POP3 host was my.mail.com and your User name was jdoe, your new User name would be my.mail.com:jdoe
  • You don't need to change the password.
  • Click the OK Button to save the changes.
  • Ensure that POPFile is running and all mail will be delivered through POPFile!
  • You'll need to make sure POPFile starts whenever Windows does. If you're using the Windows installer version of POPFile, you can drag the 'Start POPFile in Background' link from the POPFile group in the Start menu to the Start -> Programs -> Startup group. Now, Windows will load POPFile each time.

OK. Nå som vi er ferdige med dette, er det tid for å sett opp filtrene, slik at når POPFile har gjort jobben sin, så kan e-postprogrammet ditt ta over og plassere alt i riktig mappe.

Avanserte brukere har muligheten til å velge om POPfile skal legge til et [klassifikasjon] merke eller ikke i emne-linjen i e-posten. Muligheten finner du nær bunnen i siden merket 'Configuration' i web-grensesnittet til POPFile. Uansett hav du velger vil fortsatt X-Text-Classification hode bli lagt til.

Outlook

  • I Outlook velg Verktøy -> Regelveiviser... [bilde]
  • Regelveiviseren dukker opp. Klikk Ny... og velg Begynn med en ny regel. [bilde]
  • Outlook kan bruke X-Text-Classification meldingshode, som gjør at sorteringen av e-post kan skje fullstendig i bakgrunnen. Klikk Neste og velg med bestemte ord i meldingshodet.
  • Klikk så den understrekete og uthevete linken bestemte ord, og skriv inn "X-Text-Classification: " fulgt av navnet på en av bøttene (buckets) dine, klikk 'Legg til', og så 'OK'.
  • Klikk Neste og velg flytt den til mappen angitt.
  • Klikk så den understrekete og uthevete linken angitt og en mappeliste dukker opp. Velg den mappen du vil at e-posten fra denne bøtten skal sendes til og klikk 'OK'.
  • Klikk 'OK', så 'Fullfør'.
  • Gjenta prosessen for hver av bøttene dine - og klikk til slutt 'OK'.
  • Du er ferdig!
  • En liten merknad: I Web-grensesnittet for POPFile, på siden 'Configuration', kontroller at valget 'Subject Line Modification' er slått av (off).

Outlook Express

  • I Outlook Express velg Verktøy->Meldingsregler -> E-post... [bilde]
  • Vinduet for meldingsregler dukker opp. [bilde]
  • Outlook Express kan ikke bruke X-Text-Classification meldingshode, som muliggjør at sorteringen av e-post kan skje fullstendig i bakgrunnen. Så, istedenfor klikker du Hvor emnelinjen inneholder bestemte ord og klikk også Flytt den til mappen angitt [bilde]
  • Klikk den blå uthevede linken inneholder bestemte ord og dialogboksen 'Skriv inn bestemte ord' dukker opp [bilde]
  • Skriv inn navnet på en av bøttene dine, omgitt av [hakeparentes], og klikk 'OK'. Klikk så den blå uthevede linken bestemte ord og dialogboksen 'Flytt' dukker opp. elg den mappen du vil at e-posten fra denne bøtten skal sendes til og klikk 'OK'. [bilde]
  • Klikk OK.
  • Gjenta prosessen for hver av bøttene dine - og klikk til slutt 'OK'.
  • Du er ferdig!
  • En liten merknad: I Web-grensesnittet for POPFile, på siden 'Configuration', kontroller at valget 'Subject Line Modification' er slått på (on).

Eudora

(Merknad: Ettersom oversetter av manualen ikke benytter Eudora er disse instruksjonene fortsatt på engelsk)

Eudora allows filtering based on any mail header, which is great - everything can be done completely in the background, without changing the subject line!

  • Select the Tools->Filters menu. [screenshot]
  • The Filters window appears. Click New to create a new filter. [screenshot]
  • In the Header box type X-Text-Classification and in the box next to contains type the name of one of your buckets. Don't add square brackets around it. [screenshot]
  • The select Transfer To in the first Action box and select the mailbox you want to file message from this bucket into. [screenshot]
  • Repeat this for each of your buckets.
  • Close the Filters windows and say Yes when asked to save changes.
  • You're done!
  • One minor note: in the POPFile web interface, under the Configuration tab, check that the Subject Line Modification option is turned off.

Pegasus

(Merknad: Ettersom oversetter av manualen ikke benytter Pegasus er disse instruksjonene fortsatt på engelsk)

  • In Pegasus select the Tools->Mail Filtering Rules ->Edit New Mail Filtering Rules ->Rules Applied When Folder is Opened.
  • The New Mail Filtering Rules window appears.
  • Click the New Rule button. The Create a New Filtering Rule window appears.
  • Click on the Expression button. Note that using the Expression filter will work even if the POPFile Subject Line Modification option is turned off. If you use the Header filter, the POPFile Subject Line Modification option must be turned on.
  • Select the If this regular expression dialog box
  • Enter the name of one of your buckets, preceded X-Text-Classification:, for example: X-Text-Classification: spam
  • Select the Headers or body radio button
  • Select the desired action from the Action drop down list. For example Move
  • If an action such as Move or Forward has been chosen, click the Set button.
  • Select the destination folder or enter the forward to address as appropriate for the Action selected.
  • Click the OK button to add the rule to the rule list.
  • Repeat the process for each rule you need.
  • The order in which the filter rules are applied can be adjusted using the red / blue (up / down) arrows in the New Mail Filter Rules window.
  • Select the Save button to save the rules set changes.
  • Select the OK button.
  • You're done!

Dette var alt!... men...

Til tross for alt dette vil ikke POPFile sortere e-post riktig ennå. Hvorfor ikke? POPFile vet ikke hva de ulike bøttene du har opprettet betyr. POPFile har ingen formening om hva 'spam' og  'ekte' betyr, eller 'arbeid' og 'personlig' er. POPFile vet ikke engang hvilket språk e-posten din er skrevet i - og det bryr seg ikke. Du må lære POPFile å skille de ulike kategoriene e-post fra hverandre. Du gjør dette gjennom trening - og vi tar oss av dette i neste trinn.
