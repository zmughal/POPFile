جعل برنامج البريد يتكلم مع POPFile

يعمل POPFile كـملقم وكيل (بروكسي). برنامج البريد يتكلم مع POPFile، الذي بدوره يتكلم مع خادم البريد نيابة عن برنامج البريد. بدلاً من أن يستلم برنامج البريد الرسائل مباشرة، يستلمها أولاً POPFile، ويقوم بتفحصها ويقرر في أي دلو المفروض أن تدخل.

بعد أن يتم القرار - الذي يأخذ خزء من الثانية فقط - يقوم POPFile بأحد الأمرين التاليين: يستطيع إضافة بطاقة مثل [this] على بداية سطر الموضوع؛ الذي يعني أن برامج الترتيب البسيطة مثل ميكروسوفت Outlook Express تستطيع إيضال البريد إلى المكان المفروض - أو، لبرامج متقدمة مثل Eudora، يستطيع إضافة رأسية جديدة على الرسالة مثل هذه:

X-Text-Classification: personal

- الذي يستعملها برنامج البريد للترتيب.

على أية حال، يحتاج برنامج البريد لأمرين ليتكلم مع POPFile:

  1. تعديل إعدادات البرنامج لجعله يتكلم مع POPFile، و
  2. إعداد قواعد بريدية لجعل الأمور تسير حسب الازم.

لنقم بإعدادات البريد أولاً. سنقوم بإعداد القواعد لاحقاً.

ستجد في الأسفل التعليمات لإعداد برامج البريد الشائعة الإستعمال. لبرامج أخرى - أو لمستخدمين محترفين - هذه هي التعليمات العامة:

  • تعديل خادم POP3 في برنامج البريد إلى 127.0.0.1، وليس إلى الخادم الأصلي.
  • تعديل إسم مستخدم POP3 إلى [إسم خادم POP3 الأصلي]:[إسم المستخدم]
  • أترك كلمة المرور كما هي

للحسابات التي تستعمل مصادقة كلمة مرور آمنة (Secure Password Authentication), راجع إستعمال POPFile مع: مصادقة كلمة مرور آمنة (Secure Password Authentication).

أختر برنامج البريد وأتبع العمود للأسفل. بالرغم من أن هذه التلعيمات ممكن أن تبدو معقدة، إلا أنها تأخذ حوالي دقيقتين - ومن الصعب أن تخطئ. (فقط للطوارئ، قم بحفظ الإعدادات الأصلية إذا إحتجت أن تعود لها.)

Outlook

  • في Outlook إختر من القائمة أدوات->حسابات البريد الإلكتروني... . سيظهر مربع حوار حسابات البريد الإلكتروني. [لقطة]
  • إضغط على عرض حسابات البريد الإلكتروني الموجودة أو تغيرها ثم إضغط التالي.إختر الحساب المراد تعديله ليعمل مع POPFile وإضغط تغيير... قم بتسجيل إسم ملقم البريد الوارد (POP3) واسم الحساب.[لقطة]
  • قم بتغيير ملقم البريد الوارد (POP3) إلى 127.0.0.1 واسم الحساب إلى دمج اسم ملقم البريد الوارد (POP3) الأصلي واسم الحساب بفاصل بينهما هو النقطتان ":". [لقطة]
  • مثلاً، إذا كان اسم الملقم الأصلي my.mail.com واسم الحساب jdoe، فإن اسم الحساب الجديد سيصبح my.mail.com:jdoe
  • ليس هناك حاجة لتغيير كلمة المرور.
  • إضغط التالي ومن ثم إنهاء.
  • تأكد من ان POPFile قيد التنفيذ وسيتم تسليم كل البريد مروراً بـِ POPFile!
  • يجب أن تتأكد من أن POPFile يحمّل كلما أعيد تشغيل ويندوز. إذا كنت تستعمل إصدارة التنصيب السهل للنوافذ، يمكن سحب وإفلات إختصار 'Start POPFile in Background' من مجموعة POPFile إلى مجموعة تسغيل تلقائي في قائمة ابدأ. الأن سيتم تنفيذ POPFile في كل تشغيل للنوافذ.
  • ملاحظة مهمة! إذا لم يكن POPFile محمّل، فإن البريد لن يمر - وبرنامج البريد لن يستطيع الإتصال مع ملقم البريد!

  • إختر لسان ملقمات. قم بتسجيل إسم ملقم البريد الوارد (POP3) واسم الحساب. [لقطة]
  • قم بتغيير ملقم البريد الوارد (POP3) إلى 127.0.0.1 واسم الحساب إلى دمج اسم ملقم البريد الوارد (POP3) الأصلي واسم الحساب بفاصل بينهما هو النقطتان ":". [لقطة]

Eudora

  • In Eudora select the Tools->Options... menu option. The Options dialog will appear. [screenshot]
  • Make a note of the Mail Server (incoming) server name and the Login Name. [screenshot]
  • Change the Mail Server (incoming) server name to 127.0.0.1 and change the Login Name to the combination of the original Mail Server (incoming) server name and the original Login Name separated by a colon. [screenshot]
  • So, if the original Mail Server (incoming) server name was my.mail.com and your username was jdoe, your new username would be my.mail.com:jdoe
  • You don't need to change the password.
  • Hit OK.
  • Ensure that POPFile is running and all mail will be delivered through POPFile.
  • You'll need to make sure POPFile starts whenever Windows does. If you're using the Windows installer version, you can drag the 'Start POPFile in Background' link from the POPFile group in the Start menu to the Startup group. Now, Windows will load POPFile each time.
  • Important! If POPFile isn't loaded, the mail will not get through - and your email program won't connect!

Pegasus

  • In Pegasus select the Tools->Internet Options... Receiving (POP3) tab. The POP3 account setup page will appear.
  • Under General settings for receiving mail using POP3, select the POP3 Host dialog box. Make a note of the current POP3 host.
  • Change the POP3 Host to 127.0.0.1
  • Under General settings for receiving mail using POP3, select the User name dialog box. Make a note of the current User name.
  • Change the User name to the combination of the original POP3 Host name and the original User name separated by a colon.
  • So, if the original POP3 host was my.mail.com and your User name was jdoe, your new User name would be my.mail.com:jdoe
  • You don't need to change the password.
  • Click the OK Button to save the changes.
  • Ensure that POPFile is running and all mail will be delivered through POPFile!
  • You'll need to make sure POPFile starts whenever Windows does. If you're using the Windows installer version of POPFile, you can drag the 'Start POPFile in Background' link from the POPFile group in the Start menu to the Start -> Programs -> Startup group. Now, Windows will load POPFile each time.

حسناً. بعد الإنتهاء من إعداد الحساب، حان الوقت لإعداد قواعد الرسائل، حتى بعد أن يعمل POPFile عمله، يكمل برنامج البريد العمل ويضع الرسائل في المكان الصحيح.

للمستخدمين المحترفين، هناك الإمكانية لتغيير سطر الموضوع بإضافة جملة [classification] أو عدم إضافتها. هذا الخيار موجود بشكل مستقل لكل دلو في صفحة الإعدادات في واجهة المستخدم. مهما تغيّرت، ستبقى رأسية X-Text-Classification موجودة.

Outlook

  • في Outlook إختر القائمة إدوات->قواعد وتنبيهات... [لقطة]
  • سيظهر مربع حوار إعداد القواعد. إضعط قاعجة جديدة... وثم إختر البدء من قاعدة فارغة. [لقطة]
  • يستطيع Outlook التعامل مع الرأسية X-Text-Classification، التي تسمح للبريد أن يصنف كلياً في الخلفية. إضغط التالي وإختر مع كلمات معينة في رأس الرسالة. [لقطة]
  • الآن إضغط الرابط المعلم كلمات معينة وأدخل "X-Text-Classification: " متبوعاً بإسم أحد الدلاء وإضغط أضف، ثم موافق. [لقطة]
  • إضغط التالي وإختر نقل الى المجلد معين. [لقطة]
  • إضغط على الوصلة معينالمعلمة وستظهر قائمة بالمجلدات. إختر المجلد المراد نقل الرسالة إليه وإضغط موافق. [لقطة]
  • إضغط موافق، ثم إنهاء.
  • أعد العملية لكل من الدلاء، وثم أغلق النافذة.
  • لقد إنتهينا!
  • ملاحظة صغيرة: في واجهة إستخدام POPFile، تحت لسان الإعدادات، تأكد من أن خيار تعديل سطر الموضوع غير نشط.

Outlook Express

  • في Outlook Express إختر أدوات->قواعد الرسالة->بريد... [لقطة]
  • سيظهر مربع حوار قاعدة بريد جديدة. [لقطة]
  • لا يستطيع Outlook Express إستعمال رأسية X-Text-Classification، والتي تسمح لعملية الترتيب تتم كلياً في الخفية. بدلاً منها، إختر حيث يحتوي السطر "الموضوع" على كلمات معينة وأيضاً إختر نقله إلى المجلد المحدد [لقطة]
  • الآن إضغط على الوصلة على كلمات معينة الموضّحة باللون الأزرق وسيظهر مربع حوار كتابة كلمات جديدة [لقطة]
  • أدخل إسم أحد الدلاء، [محاطاً بالأقواس المربعة]، ومن ثم إضغط موافق. الآن إضغط على الوصلة المحدد الموضّحة باللون الأزرق وسيظهر مربع حوار نقل. إضغط على المجلد المطلوب نقل رسائل ذلك الدلو إليه وإضغط موافق. [لقطة]
  • إضغط موافق.
  • أعد العملية لكل من الدلاء، وثم أغلق النافذة.
  • لقد إنتهينا!
  • ملاحظة صغيرة: في واجهة إستخدام POPFile، تحت لسان الإعدادات، تأكد من أن خيار تعديل سطر الموضوع نشط.

Eudora

Eudora allows filtering based on any mail header, which is great - everything can be done completely in the background, without changing the subject line!

  • Select the Tools->Filters menu. [screenshot]
  • The Filters window appears. Click New to create a new filter. [screenshot]
  • In the Header box type X-Text-Classification and in the box next to contains type the name of one of your buckets. Don't add square brackets around it. [screenshot]
  • The select Transfer To in the first Action box and select the mailbox you want to file message from this bucket into. [screenshot]
  • Repeat this for each of your buckets.
  • Close the Filters windows and say Yes when asked to save changes.
  • You're done!
  • One minor note: in the POPFile web interface, under the Configuration tab, check that the Subject Line Modification option is turned off.

Pegasus

  • In Pegasus select the Tools->Mail Filtering Rules ->Edit New Mail Filtering Rules ->Rules Applied When Folder is Opened.
  • The New Mail Filtering Rules window appears.
  • Click the New Rule button. The Create a New Filtering Rule window appears.
  • Click on the Expression button. Note that using the Expression filter will work even if the POPFile Subject Line Modification option is turned off. If you use the Header filter, the POPFile Subject Line Modification option must be turned on.
  • Select the If this regular expression dialog box
  • Enter the name of one of your buckets, preceded X-Text-Classification:, for example: X-Text-Classification: spam
  • Select the Headers or body radio button
  • Select the desired action from the Action drop down list. For example Move
  • If an action such as Move or Forward has been chosen, click the Set button.
  • Select the destination folder or enter the forward to address as appropriate for the Action selected.
  • Click the OK button to add the rule to the rule list.
  • Repeat the process for each rule you need.
  • The order in which the filter rules are applied can be adjusted using the red / blue (up / down) arrows in the New Mail Filter Rules window.
  • Select the Save button to save the rules set changes.
  • Select the OK button.
  • You're done!

هذا هو كل شيء! ... ولكن...

حتى مع كل هذا، فإن POPFile لن يقوم بتصنيف وترتيب البريد الآن. لما لا؟ إنه لا يعرف معنى أي من الدلاء المعرّفة. ليس لديه أية فكرة عن ماذا يعني 'spam' أو 'normal'، أو ما هو 'work' أو 'personal'. حتى إنه لا يعرف بأية لغة مكتوب البريد - وهذا لا يهمه. تحتاج إلى تدريبه للتمييز بين البريد العادي والبريد الخردة، أو بين مشروع وآخر. يتم عمل هذا بـِالتدريب - و سيتم مناقشته في القسم التالي.
