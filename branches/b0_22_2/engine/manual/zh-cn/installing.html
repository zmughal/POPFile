安装 POPFile

POPFile 有两种版本 - 一种是容易安装的 Windows 版, 另一种是给技术人员安装的跨平台版. 不过装好之后, 这两个版本将会一模一样.

容易安装的 Windows 版

从 POPFile 下载页面取得 Windows 安装档, 然后执行 setup.exe 并按照那里的指示继续下去. 你可以从开始选单 (开始 > 程序 > POPFile > Run POPFile in Background) 来激活 POPFile, 或者重新开机后她就会自动跑起来. 重要: 啥也不会出现在你的屏幕上! 如果你想要检查看看她是否正确地被执行了, 请阅读底下的 "存取 POPFile" 段落.

系统需求

你需要一些东西才能让 POPFile 开始动起来.

  1. 最新释出的 POPFile.
  2. 使用 POP3 通讯协议的电子邮件帐号 (多数的帐号都是; 不过如果没有额外的软体相助, 你将不能搭配 POPFile 一并使用像是 Hotmail 和 Yahoo! Mail 这类的网页邮件服务)
  3. 大约 2MB 左右的硬碟空间

你可以在 POPFile 下载页面套件清单的最上面取得最新释出的 POPFile 的 zip 格式压缩档. Windows 使用者祇会想要抓被放在 Windows 档区里的档案.

POPFile 本身祇会占据大约 1 MB 再多一点空间而已. 用来分类电子邮件的字表 (又被称做尸体) 还会占掉额外的一些空间, 不过这会依你如何调教 POPFile, 以及你所建立的邮筒数量而有所不同.

跨平台版

首先要在你的机器上把 Perl 给跑起来, 然后从 POPFile 首页下载 POPFile Perl 脚本的 zip 压缩档, 并解开到你所指定的目录里. 在 Windows 平台上的话, 就用 perl popfile.pl 这个指令来加以激活.

检查 POPFile 是否正确地跑起来了

POPFile 没有多数程序的那种传统接口 - 你其实是用你自己的网页浏览器来对她加以存取的. 别担心 - 除非你首肯, 否则你机器外的任何人都没办法碰到她.

要加载 POPFile 的话, 请连到 http://127.0.0.1:8080, 或者 (对 Windows 版的使用者来说) 在开始选单里的 'POPFile Web Interface' 按一下. 重要: 如果, 在 Windows 的安装过程中, 你选择要用其它的连接埠来跟 POPFile 交谈的话, 请把这里的 "8080" 换成任何你所挑选的数字.

如果你得到错误讯息的话, 请先检查 POPFile 是否真的已经跑起来了, 然后看看连接埠号码是否正确 (她应该要是 8080 才对, 但是如果你曾变更过, 那么你就得一并变更你所要连的网址.)

如果你看到 POPFile 画面的话, 恭喜啦! 现在你可以开始设定要如何分类邮件了. 让咱们继续设定邮筒吧. >>
